## v3.3.3 — 2026-07-21 — 本草本地优先查询规则

### 变更内容

**SKILL.md修改**：
1. **工作流顶部新增"本地优先原则"**
   - 查药物性味、方剂组成、穴位定位等，必须先查本地 references/ 文件，禁止直接联网搜索
   - 明确本地资源覆盖范围：356味药（本草经索引）、16部古籍、74+医案、针灸处方手册
   - 仅当本地文件确实无相关内容时，才可联网补充
2. **Step 4（选方）新增"本草查询"子步骤**
   - 第1步：Grep 搜索 `本草经_药名索引.md` 定位品级+文件
   - 第2步：Read 对应上品/中品/下品文件获取详细性味主治
   - 第3步：如需更深入注释 → 查本草正义或本草备要
   - 标注禁止联网查药性
3. **药物→本草模块路由表标题强化**：增加"⚠️ 必须本地查询，禁止联网搜药性"

### 设计意图

确保所有需要查本草的场景（选方、加减、剂量决策、药物知识问答等）都走本地索引，避免联网查询导致响应慢或引入不可靠信息。

## v3.3.2 — 2026-07-20 — 工作流新增医案查询步骤

### 变更内容

**SKILL.md修改**：
1. **临床辨证工作流新增 Step 4.5：医案查询与验证**
   - 位于 Step 4（选方）和 Step 4.6（剂量决策）之间
   - 根据六经归属+证型+症状，查询医案库6个分类文件寻找相似案例
   - 查询方法：Grep 搜索证型关键词
   - 匹配结果处理：一致度高→参考药物加减 / 一致度低→重新审视 / 无匹配→注明无参考
   - 原 Step 4.5（剂量决策）顺延为 Step 4.6
2. **辨证输出格式新增"参考医案"字段**
   - 位于"治法方药"段落，"加减"之后
   - 有相似医案时注明出处，无则标注"无相似医案参考"

### 设计意图

将医案查询从隐式行为变为显式工作流步骤，确保每次辨证都经过医案验证环节，提升选方的临床可靠性。

## 进化规则

### 版本号规则

- **vX.0**（大版本）：结构性变更，如新增整个章节、改变辨证框架
- **vX.Y**（小版本）：内容充实，如增加方剂、补充医案、修正错误
- 每次更新记录在本文档中

### 更新触发条件

1. **辨证错误**：发现辨证路径有误，需修正
2. **内容遗漏**：发现重要内容未收录
3. **用户反馈**：用户报告使用中的问题
4. **知识更新**：从古籍中发现新的有价值内容
5. **结构优化**：发现更好的组织方式

### 更新流程

```
发现问题 → 查阅古籍原文确认 → 修改SKILL.md → 记录本日志 → 验证效果
```

### 质量门控

每次更新后检查：
- [ ] 修改内容是否有古籍原文支持？
- [ ] 是否保持了经方派的核心立场？
- [ ] 辨证路径是否清晰可操作？
- [ ] 是否添加了不必要的争议内容？
- [ ] 引用路径是否正确？
